# SDD Process — Workflow Router

Use this guide to decide which workflow applies before starting work.

## Workflow Router

| Type of change | Skill | Purpose |
|---|---|---|
| New feature, API change, data model change | `/grill-with-docs` → `/to-prd` → `/to-issues` | Capture requirements, generate PRD, decompose into vertical issues |
| Architectural change (new module/layer) | `/grill-with-docs` → `/improve-codebase-architecture` | Evaluate design, propose refactoring |
| Bug fix with identified cause | `/tdd-cycle` | Red-Green-Refactor cycle with Engram saves |
| Testable improvement (refactor with behavior) | `/tdd-cycle` | TDD-driven refactoring |
| Code review | `/review` | Correctness, quality, security, coverage |
| Quality/security audit | `/audit` | Dependencies, linters, secrets, patterns, tests |
| Git workflow (commit/push/PR) | `/git-flow` | Semantic commit, push, optional PR |
| Rename, move, refactor (no behavior change) | Direct (plan + approval) | No skill needed, standard permission protocol |
| Exploration / prototype | `scratch/` folder (gitignored) | Throwaway code; re-enter via grill-with-docs if useful |
| Step back and evaluate | `/zoom-out` | Assess architecture and design decisions |

## Issue Tracking (Local Markdown)

All issues live under `.scratch/<feature>/`:

- **PRD**: `.scratch/<feature>/PRD.md` — generated by `/to-prd`
- **Issues**: `.scratch/<feature>/issues/<NN>-<slug>.md` — generated by `/to-issues`
- **INDEX.md**: `.scratch/<feature>/INDEX.md` — contains dependency graph (ASCII) + status table

### INDEX.md Format

**Dependency graph** (ASCII):
```
[issue-01] ──> [issue-02]
             ╱ [issue-03]
```

**Status table**:

| ID | Slug | Status | Summary |
|---|---|---|---|
| 01 | feature-auth | `ready` | Implement JWT auth |
| 02 | feature-login-form | `blocked` | Blocked on issue-01 |
| 03 | feature-logout | `ready` | Add logout endpoint |

### Status Values

- `ready` — dependencies met; can be started
- `blocked` — waiting on another issue
- `in-progress` — actively being worked
- `done` — completed and merged

### Agent Workflow

1. Agent reads INDEX.md
2. Picks all `ready` issues
3. Launches them in parallel via `/tdd-cycle`
4. `/tdd-cycle` updates INDEX.md on completion (status → `done`)

### Directory Structure

```
.scratch/
└── my-feature/
    ├── PRD.md
    ├── INDEX.md
    └── issues/
        ├── 01-feature-auth.md
        ├── 02-feature-login-form.md
        └── 03-feature-logout.md
```
